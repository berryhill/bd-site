        <section id="what"><h2>What I actually do</h2><p>The honest answer is: <b>I build and inspect AI-native operating loops.</b></p><p><b>Engineering.</b> I work on the code, review gates, publishing surfaces, and glue that make agent work auditable instead of magical.</p><p><b>Writing.</b> I use essays as field notes: what broke, what shipped, what needed a human decision, and where the agent loop produced real leverage.</p><p><b>Strategy.</b> I care about capital markets, crypto rails, AI/ML systems, and automation only when the discussion gets specific enough to change a build decision.</p></section>
        <section id="beliefs"><h2>Things I believe</h2><p>Not a manifesto. A fast filter for the kind of work this site takes seriously.</p><div class="belief"><div class="num"><b>01</b></div><div class="text"><b>Agentic development is real, but the durable work is governance.</b> The useful question is not whether an agent can produce output. It is who reviews it, what persists, what escalates, and how the next run gets better.</div></div><div class="belief"><div class="num"><b>02</b></div><div class="text"><b>Provenance beats vibes.</b> If a workflow cannot show its sources, handoffs, assumptions, and review trail, it is not ready to carry serious decisions.</div></div><div class="belief"><div class="num"><b>03</b></div><div class="text"><b>The operator loop is the product.</b> Tools matter, but the advantage comes from disciplined decomposition, verification, memory, and escalation.</div></div><div class="belief"><div class="num"><b>04</b></div><div class="text"><b>Specificity is the cheapest leverage.</b> Name the constraint, the artifact, the gate, and the failure mode. Otherwise the copy is probably hiding weak thinking.</div></div></section>

        <section id="working"><h2>How I work with people</h2><p>The shape of serious work is roughly the same:</p><ul><li>Start with the real constraint, not the deck constraint.</li><li>Map what needs to be built, proven, measured, or explained.</li><li>Ship small artifacts that make the next decision less fake.</li><li>Keep the operator loop visible.</li></ul></section>
